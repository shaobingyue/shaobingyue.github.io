JavaScript 核心概念：原型、执行上下文与闭包

在 JavaScript 中，原型、执行上下文、闭包是三个贯穿始终的核心概念，也是面试高频考点。本文会从“是什么-怎么用-为什么”三个维度，结合代码示例拆解这些概念，帮你彻底理解并落地到实际开发中。

一、原型与原型链

原型是 JavaScript 实现继承的核心机制，所有对象（除 null 外）都有隐式原型，函数有显式原型，二者共同构成原型链。

1.1 显式原型（prototype）与隐式原型（proto）

  • 显式原型：每个函数自带 prototype 属性，是一个对象，用于存储“所有实例共享的属性/方法”。
  • 隐式原型：每个对象（实例）自带 __proto__ 属性（浏览器环境），指向其构造函数的 prototype。

代码示例：原型的关联关系

// 1. 定义构造函数
function Person(name) {
  this.name = name; // 实例自身属性
}

// 2. 在显式原型上定义共享方法（所有实例可复用）
Person.prototype.sayHi = function() {
  console.log(`Hi, I'm ${this.name}`);
};

// 3. 创建实例
const person1 = new Person("张三");
const person2 = new Person("李四");

// 4. 验证原型关联
console.log(person1.__proto__ === Person.prototype); // true（实例隐式原型 → 构造函数显式原型）
console.log(Person.prototype.constructor === Person); // true（原型的 constructor 指向构造函数）

// 5. 调用共享方法（实例自身没有，沿原型链查找）
person1.sayHi(); // 输出：Hi, I'm 张三
person2.sayHi(); // 输出：Hi, I'm 李四
